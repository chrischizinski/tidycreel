**1. Add warn_tier2_interview_issues() to R/survey-bridge.R:**

Add after warn_tier2_group_issues(). Follow the warn_tier2_issues() pattern:

```r
#' Validate interview data quality (Tier 2)
#'
#' Internal function that checks for data quality issues in interview data
#' attached to a creel_design. Issues warnings (not errors) for suspicious
#' values. These are Tier 2 checks - data quality issues that should be
#' investigated but don't prevent estimation.
#'
#' @param design A creel_design object with interviews attached
#'
#' @return NULL (invisible) - function called for side effects (warnings)
#'
#' @keywords internal
#' @noRd
```

Checks to implement (all use cli::cli_warn, same bullet style as warn_tier2_issues):

a. **Very short trips** (effort < 0.1 hours = 6 minutes):
   - Only check if effort_col is not NULL
   - Count: `sum(interviews[[effort_col]] < 0.1, na.rm = TRUE)`
   - Message: "{n} interview{?s} ha{?s/ve} effort < 0.1 hours (6 minutes)."
   - Hint: "Very short trips may indicate data entry errors."

b. **Zero catch values:**
   - Count: `sum(interviews[[catch_col]] == 0, na.rm = TRUE)`
   - Message: "{n} interview{?s} ha{?s/ve} zero catch."
   - Hint: "Zero catch may be valid (skunked) or indicate missing data."

c. **Negative catch values:**
   - Count: `sum(interviews[[catch_col]] < 0, na.rm = TRUE)`
   - Message: "{n} interview{?s} ha{?s/ve} negative catch values."
   - Warning bullet: "Negative catch indicates data entry errors."
   - Hint: "Review and correct before estimation."

d. **Negative effort values:**
   - Only check if effort_col is not NULL
   - Count: `sum(interviews[[effort_col]] < 0, na.rm = TRUE)`
   - Message + warning similar to negative catch

e. **Missing effort values (NA):**
   - Only check if effort_col is not NULL
   - Count: `sum(is.na(interviews[[effort_col]]))`
   - Message: "{n} interview{?s} ha{?s/ve} missing effort values."
   - Hint: "Missing effort limits CPUE estimation."

f. **Sparse interview coverage per stratum** (< 3 interviews per stratum):
   - Create strata variable same way as warn_tier2_issues (single vs interaction)
   - Count interviews per stratum
   - If any stratum has < 3: warn with stratum-specific bullets (same format as sparse strata warning in warn_tier2_issues)

```r
# Example dataset integration tests ----

test_that("example_interviews works with example_calendar in complete workflow", {
  data(example_calendar, package = "tidycreel")
  data(example_interviews, package = "tidycreel")

  design <- creel_design(example_calendar, date = date, strata = day_type)
  result <- add_interviews(design, example_interviews,
                           catch = catch_total,
                           effort = hours_fished,
                           harvest = catch_kept)

  expect_s3_class(result, "creel_design")
  expect_false(is.null(result$interviews))
  expect_false(is.null(result$interview_survey))
  expect_equal(result$catch_col, "catch_total")
  expect_equal(result$effort_col, "hours_fished")
  expect_equal(result$harvest_col, "catch_kept")
  expect_equal(result$interview_type, "access")
})

test_that("example_interviews has expected structure", {
  data(example_interviews, package = "tidycreel")

  expect_s3_class(example_interviews, "data.frame")
  expect_true(nrow(example_interviews) > 0)
  expect_true("date" %in% names(example_interviews))
  expect_true("hours_fished" %in% names(example_interviews))
  expect_true("catch_total" %in% names(example_interviews))
  expect_true("catch_kept" %in% names(example_interviews))
  expect_true(inherits(example_interviews$date, "Date"))
  expect_true(is.numeric(example_interviews$hours_fished))
  expect_true(is.numeric(example_interviews$catch_total))
  expect_true(is.numeric(example_interviews$catch_kept))
  # Validate catch_kept <= catch_total
  expect_true(all(example_interviews$catch_kept <= example_interviews$catch_total))
})
```
