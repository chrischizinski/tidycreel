**B. Zero-effort handling in estimate_cpue_total() and estimate_cpue_grouped():**

In both functions, after retrieving interviews_data and before building formulas, add zero-effort filtering:

```r
# Filter out zero-effort interviews with warning
zero_effort <- !is.na(interviews_data[[effort_col]]) & interviews_data[[effort_col]] == 0
if (any(zero_effort)) {
  n_zero <- sum(zero_effort)
  cli::cli_warn(c(
    "{n_zero} interview{?s} with zero effort excluded from CPUE estimation.",
    "i" = "CPUE requires effort > 0 (catch/effort is undefined for effort = 0)."
  ))
  interviews_data <- interviews_data[!zero_effort, , drop = FALSE]
}
```

For the estimate_cpue_total function, use the filtered interviews_data to reconstruct the survey design:
- After filtering, need to create a temporary design with filtered interviews to pass to svyratio

- Then apply get_variance_design() to that temporary design
- The n in result should reflect filtered count: `nrow(interviews_data)` after filtering

For estimate_cpue_grouped, same pattern: filter first, then build temporary survey design from filtered data.

NOTE: Only apply this filtering if zero-effort rows actually exist. If no zeros, proceed normally using design$interview_survey directly (most common case -- no performance penalty for clean data).
