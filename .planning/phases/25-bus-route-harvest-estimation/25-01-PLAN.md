**`estimate_harvest_br(design, by_vars, variance_method, conf_level, verbose, use_trips)`**

Formula: H_hat = sum(h_i / pi_i) where h_i = harvest_col * .expansion (Jones & Pollock 2012, Eq. 19.5)

Implementation steps:
1. If `verbose = TRUE`, emit `cli::cli_inform("Using bus-route estimator (Jones & Pollock 2012, Eq. 19.5)")`.
2. Retrieve `interviews <- design$interviews`.
3. Defensive check: `.expansion` must be in `names(interviews)` — hard error with `cli::cli_abort` directing to `add_interviews(..., n_counted=, n_interviewed=)`.
4. Defensive check: `.pi_i` must be in `names(interviews)` — hard error with `cli::cli_abort`.
5. Check for `NA` in `interviews$.pi_i` — hard error listing site+circuit combos (copy pattern from `estimate_effort_br()`).
6. Apply `use_trips` filtering:
   - `use_trips = "complete"`: filter `interviews` to rows where `design$trip_status_col` is non-NULL AND the trip_status column value is `"complete"` (case-insensitive). If `design$trip_status_col` is NULL, skip filtering (backward-compat).
   - `use_trips = "incomplete"`: filter to incomplete trips, then apply pi_i-weighted MOR (per-site mean-of-ratios with pi_i weighting): compute `h_ratio_i = harvest_col / effort_col` for each incomplete interview row, then `contribution_i = h_ratio_i / .pi_i`. The HT estimate is `sum(contribution_i)`. Use same survey::svytotal on `.contribution` column approach as effort estimator.
   - `use_trips = "diagnostic"`: run both complete (Eq. 19.5) and incomplete (pi_i-weighted MOR) paths; return `creel_estimates_diagnostic` class with both results as components `$complete` and `$incomplete`. Return early from the function after building diagnostic object.
7. Compute `interviews$.h_i <- interviews[[harvest_col]] * interviews$.expansion`.
8. Handle zero-count sites (n_counted=0, n_interviewed=0): set `.h_i` to 0 for those rows (same pattern as `.e_i` in effort estimator — check `n_counted_col` and `n_interviewed_col` from design).
9. Compute `interviews$.contribution <- interviews$.h_i / interviews$.pi_i`.
10. Build per-site breakdown named list keyed by species. The harvest column is `design$harvest_col`. Group by species if `by_vars` includes a species column; otherwise build a single-element list: `list(harvest = data.frame(site, circuit, h_i, pi_i, h_i_over_pi_i))`. Use `design$bus_route$site_col` and `design$bus_route$circuit_col` for site/circuit columns.
    - The species-keyed list structure: if there is no species grouping, use `"total"` as the list key. The attribute name on the result is `"site_contributions"` (same as effort — `get_site_contributions()` already handles both).
11. For ungrouped estimation: compute `total_estimate <- sum(interviews$.contribution, na.rm = TRUE)`. Build `survey::svydesign` on interviews data. Apply `get_variance_design()`. Run `survey::svytotal(~.contribution, svy_br)`. Extract estimate, SE, CI. Build `estimates_df` tibble with `estimate, se, ci_lower, ci_upper, n`. Call `new_creel_estimates(estimates = estimates_df, method = "total", variance_method, design, conf_level, by_vars = NULL)`. Attach `site_contributions` attribute.
12. For grouped estimation (`by_vars` not NULL): follow exact grouped pattern from `estimate_effort_br()` — `survey::svyby()` with `FUN = survey::svytotal`, add `proportion` column (group estimate / overall total), merge per-group `n`. Return `new_creel_estimates()` with `site_contributions` attribute.
