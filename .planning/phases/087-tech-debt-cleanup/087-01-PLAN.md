### Task 3 — WARNING-02: Guard Schnabel ci_hi against lo_m = 0

**File:** `R/creel-estimates-mark-recapture.R`

When `sum_m` is very small (e.g., 1), `qpois(0.025, 1) = 0`, so `ci_hi <- sum_Mn / lo_m` produces `Inf` silently. Guard with:

```r
ci_hi <- if (lo_m > 0) sum_Mn / lo_m else Inf
```

Or better — emit a `cli_warn()` when `lo_m == 0` so the user knows:

```r
if (lo_m == 0L) {
  cli::cli_warn("Schnabel Poisson CI: lower Poisson quantile is 0; ci_hi set to Inf.")
  ci_hi <- Inf
} else {
  ci_hi <- sum_Mn / lo_m
}
```

Add a test in `tests/testthat/test-estimate-angler-n.R`:
- Input that produces `sum_m = 1` (e.g., `M = c(0, 10)`, `n = c(5, 5)`, `m = c(0, 1)`)
- Expect `ci_hi == Inf` and the warning fires

### Task 5 — CR-01: Fix .imputed logic in impute_camera_counts()

**File:** `R/impute-camera-counts.R`

Current logic marks `.imputed = TRUE` based on post-imputation inference:
```r
status_col != "operational" & !is.na(result[[count_col]])
```

This produces a false positive when a non-operational row already had a non-NA count before imputation (biologist manually keyed a value).

Fix: Capture a pre-imputation NA baseline before the per-stratum loop, then use it to set `.imputed`:

```r
# Before the loop:
original_na <- is.na(data[[count_col]])

# After the loop, replace the .imputed assignment:
result[[".imputed"]] <- original_na & !is.na(result[[count_col]])
```

This correctly identifies only rows that were NA before and non-NA after.

Add a test in `tests/testthat/test-impute-camera-counts.R`:
- A non-operational row that already has a non-NA count
- Expect `.imputed == FALSE` for that row

**Verify:** `devtools::test(filter = "impute")` passes with 0 failures.
