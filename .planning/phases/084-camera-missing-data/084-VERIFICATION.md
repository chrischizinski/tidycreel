**Two code-review warnings carried forward (not blocking):**

- **CR-01** — `.imputed` false positive for non-operational rows with pre-existing non-NA counts. Recommend: snapshot `is_outage` before the per-stratum loop and use that vector (rather than post-imputation inference) to set `.imputed`.
- **CR-02** — Docs say "ZINB" but `glmmTMB::nbinom2()` without `ziformula` is plain NB GLMM. Recommend: correct docstring to "negative binomial GLMM" or add `ziformula = ~1` to achieve the stated ZINB behaviour.
