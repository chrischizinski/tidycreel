```r
#' Impute missing camera counts using GLM or GLMM
#'
#' `r lifecycle::badge("experimental")`
#'
#' @description
#' Fills outage rows in a camera count data frame using a per-stratum model.
#' The GLM method (default, Hartill 2016) fits a Poisson GLM with day-type
#' as the sole predictor. The GLMM method (Afrifa-Yamoah 2020) fits a
#' zero-inflated negative binomial GLMM and requires the `glmmTMB` package
#' (in `Suggests`).
#'
#' Outage rows are identified as any row where `status_col != "operational"`
#' AND `count_col` is `NA`. All rows are returned; imputed rows have
#' `.imputed = TRUE`.
```

**@param block:** Document all six arguments per D-01 — `data`, `count_col`, `strata_col`, `status_col = "camera_status"`, `method = "glm"`, `site_col = NULL`.

**@return:** Data frame with same columns as input plus `.imputed` (logical). `count_col` is integer; outage rows are filled. Row count equals `nrow(data)`.

**@references:** Hartill, B.W., Cryer, M., and Morrison, M.A. 2020 (doi:10.1016/j.fishres.2020.105706); Afrifa-Yamoah, E., Mueller, U.A., Taylor, S.M., and Fisher, A. 2020.

**@examples** block (`\dontrun{}`): Show the full chain — `data(example_camera_counts)`, call `impute_camera_counts(example_camera_counts, count_col = "ingress_count", strata_col = "day_type")`, then pass to `add_counts()`.

1. **Input validation** (per D-01, using checkmate — already in Imports):
   ```r
   checkmate::assert_data_frame(data, min.rows = 1L)
   checkmate::assert_string(count_col)
   checkmate::assert_string(strata_col)
   checkmate::assert_string(status_col)
   checkmate::assert_choice(method, c("glm", "glmm"))
   checkmate::assert_string(site_col, null.ok = TRUE)
   ```
   Then check columns exist in data — if count_col, strata_col, or status_col are absent, abort with `cli::cli_abort(c("Column {.field {col}} not found in {.arg data}.", "i" = "Available columns: {.field {names(data)}}"))`.

2. **GLMM guard** (per D-12 — canonical pattern from creel-estimates-aerial-glmm.R:95):
   ```r
   if (method == "glmm") {
     rlang::check_installed(
       "glmmTMB",
       reason = "to fit zero-inflated negative binomial GLMM for camera count imputation"
     )
   }
   ```

3. **Identify outage rows** (per D-02):
   ```r
   is_outage <- data[[status_col]] != "operational" & is.na(data[[count_col]])
   ```

4. **Missing fraction per stratum and high-missingness warning** (per D-15 / CAMP-04):
   Compute for each unique stratum value: `sum(is_outage[stratum_rows]) / length(stratum_rows)`.
   After computing all fractions, fire one warning if any > 0.5:
   ```r
   if (length(high_miss_strata) > 0L) {
     cli::cli_warn(c(
       "High missingness detected in {length(high_miss_strata)} stratum{?/a}.",
       "!" = "{.val {high_miss_strata}} {?has/have} > 50% missing camera counts.",
       "i" = "Imputation results may be unreliable (Afrifa-Yamoah 2020)."
     ))
   }
   ```

5. **Per-stratum loop** — `lapply()` over `unique(data[[strata_col]])`:
   For each stratum value `s`:
   a. Subset to stratum rows: `stratum_data <- data[data[[strata_col]] == s, , drop = FALSE]`
   b. Identify observed rows within stratum: `obs_mask <- stratum_data[[status_col]] == "operational" & !is.na(stratum_data[[count_col]])`
   c. **All-missing abort** (per D-16 / CAMP-05):
      ```r
      if (!any(obs_mask)) {
        cli::cli_abort(c(
          "Stratum {.val {s}} has no observed counts.",
          "x" = "All {nrow(stratum_data)} rows in stratum {.val {s}} are outages or NA.",
          "i" = "Imputation requires at least one operational observation per stratum."
        ))
      }
      ```
   d. If no outage rows in this stratum, return `stratum_data` unchanged.
   e. Build formula: `formula <- stats::as.formula(paste0(count_col, " ~ ", strata_col))`
      Note: `strata_col` is the day_type column per D-10. When fitting within a stratum, day_type has one unique value so the formula reduces to an intercept-only GLM for that stratum, which is correct — the prediction for any new row in that stratum is the stratum mean. This is intentional per D-09/D-10.
   f. **GLM path**:
      ```r
      fit <- stats::glm(formula, data = stratum_data[obs_mask, , drop = FALSE],
                        family = stats::poisson())
      ```
   g. **GLMM path** (method == "glmm"): Build formula with random effect when site_col is not NULL:
      ```r
      if (!is.null(site_col)) {
        glmm_formula <- stats::as.formula(
          paste0(count_col, " ~ ", strata_col, " + (1 | ", site_col, ")")
        )
      } else {
        glmm_formula <- stats::as.formula(paste0(count_col, " ~ ", strata_col))
      }
      fit <- tryCatch(
        glmmTMB::glmmTMB(
          glmm_formula,
          data = stratum_data[obs_mask, , drop = FALSE],
          family = glmmTMB::nbinom2(link = "log")
        ),
        error = function(e) {
          cli::cli_warn(c(
            "GLMM convergence failed for stratum {.val {s}}.",
            "i" = "Falling back to Poisson GLM for this stratum.",
            "!" = "Error: {e$message}"
          ))
          NULL
        }
      )
      if (is.null(fit)) {
        fit <- stats::glm(formula, data = stratum_data[obs_mask, , drop = FALSE],
                          family = stats::poisson())
      }
      ```
   h. **Predict for outage rows** (per D-11):
      ```r
      outage_mask <- !obs_mask & (stratum_data[[status_col]] != "operational" & is.na(stratum_data[[count_col]]))
      predicted <- predict(fit, newdata = stratum_data[outage_mask, , drop = FALSE],
                           type = "response")
      stratum_data[[count_col]][outage_mask] <- as.integer(round(predicted))
      ```
   i. Return `stratum_data`.

6. **Bind results** (per shared pattern):
   ```r
   result <- do.call(rbind, imputed_rows)
   row.names(result) <- NULL
   ```

7. **Add .imputed flag** (per D-06):
   ```r
   result$.imputed <- FALSE
   result$.imputed[is_outage[match(rownames(result_original), ...)] ] <- TRUE
   ```
   Simpler approach: track outage row indices before the loop, then after binding set `.imputed` based on whether `status_col != "operational" & !is.na(result[[count_col]])` (now filled) but originally was outage. Best implementation: initialize `.imputed = FALSE` on result, then set TRUE where `result[[status_col]] != "operational"` AND `!is.na(result[[count_col]])` AND the original data had NA there. Use: rebuild `is_outage` on the original data and match rows by position (row binding preserves stratum order within strata, so final row order matches original input order after sorting by strata then re-ordering). Cleanest: before the loop, record `original_is_outage <- is_outage` on `data`; after rbind, since lapply processes strata in order of `unique(data[[strata_col]])` and each stratum preserves its original row order, reconstruct `is_outage_final` by applying the same logic to result. Set: `result$.imputed <- result[[status_col]] != "operational" & !is.na(result[[count_col]])`. This works because outage rows had NA before imputation, and operational rows had values — so any non-operational row that now has a non-NA count was imputed.

8. **Integer coercion** (per D-08, storage.mode pattern from power-sample-size.R:71):
   ```r
   storage.mode(result[[count_col]]) <- "integer"
   ```

9. **Return** `result` (invisibly or visible — use visible return as biologists will inspect it).
