<feature>
  <name>Grouped estimation via by parameter</name>
  <files>R/creel-estimates.R, R/survey-bridge.R, tests/testthat/test-estimate-effort.R, man/estimate_effort.Rd, NAMESPACE</files>
  <behavior>
    estimate_effort() gains a `by = NULL` parameter accepting bare column names or tidyselect helpers.

    ## Cases (input -> expected output):

    ### Ungrouped (backward compat):
    estimate_effort(design)
    -> Same as Phase 4: single-row tibble with estimate/se/ci_lower/ci_upper/n

    ### Single grouping variable:
    estimate_effort(design, by = day_type)
    -> tibble with columns: day_type, estimate, se, ci_lower, ci_upper, n
    -> One row per unique value of day_type
    -> estimate/se/ci match survey::svyby(~effort, ~day_type, design$survey, svytotal)
    -> n = count of observations per group in design$counts

    ### Multiple grouping variables:
    estimate_effort(design, by = c(day_type, location))
    -> tibble with columns: day_type, location, estimate, se, ci_lower, ci_upper, n
    -> One row per combination of day_type x location
    -> Estimates match survey::svyby(~effort, ~day_type + location, design$survey, svytotal)

    ### Tidyselect helpers:
    estimate_effort(design, by = starts_with("day"))
    -> Resolves tidyselect expression against design$counts columns
    -> Same behavior as explicit column names

    ### Error cases:
    estimate_effort(design, by = nonexistent_column) -> tidyselect error about column not found
    estimate_effort(design, by = effort_hours) -> should work (any column is valid for grouping)

    ### Tier 2 group-level validation:
    When by != NULL, warn if any group has fewer than 3 observations (sparse group warning)

    ### creel_estimates attributes:
    - result$method = "total" (still total estimation, but grouped)
    - result$by_vars = character vector of grouping column names (NULL when ungrouped)
    - result$variance_method = "taylor"

    ### Print method:
    When by_vars is not NULL, format.creel_estimates shows "Grouped by: day_type" header line
  </behavior>
  <implementation>
    ## Step 1: Modify estimate_effort() signature

    Add `by = NULL` parameter. Use rlang::enquo(by) to capture expression.
    Route: if quo_is_null(by_quo), call existing ungrouped path. Otherwise, resolve
    by_vars via tidyselect::eval_select() against design$counts, then call
    estimate_effort_grouped().

    ## Step 2: Create estimate_effort_grouped() internal function

    1. Build survey formulas: count_formula = reformulate(count_var), by_formula = reformulate(by_vars)
    2. Call survey::svyby(count_formula, by_formula, design$survey, survey::svytotal, vartype = c("se", "ci"), ci.level = conf_level, keep.names = TRUE) — wrapped in suppressWarnings for expected survey package warnings
    3. Extract result columns:
       - Group columns: svy_result[, by_vars, drop = FALSE]
       - estimate: svy_result[[count_var]]
       - se: svy_result[[paste0("se.", count_var)]]
       - ci_lower: svy_result[[paste0("ci_l.", count_var)]]
       - ci_upper: svy_result[[paste0("ci_u.", count_var)]]
    4. Calculate per-group sample sizes from design$counts using table() or tapply()
    5. Combine into tibble: group_cols, estimate, se, ci_lower, ci_upper, n
    6. Preserve original column types from design$counts (factors stay factors)

    ## Step 3: Update new_creel_estimates constructor

    Add optional `by_vars = NULL` parameter to new_creel_estimates(). Store in object.

    ## Step 4: Update format.creel_estimates

    When x$by_vars is not NULL, add "Grouped by: {by_vars}" line after Variance line.

    ## Step 5: Extend Tier 2 validation for groups

    Add warn_tier2_group_issues(design, by_vars) internal function in survey-bridge.R.
    Counts observations per group. Warns if any group has < 3 observations.
    Called from estimate_effort_grouped() after resolving by_vars.

    ## Step 6: Update roxygen documentation

    Update @param by, @return, @details, @examples in estimate_effort().
    Run roxygen2::roxygenise() to regenerate man/estimate_effort.Rd and NAMESPACE.

    ## Key patterns to follow:
    - suppressWarnings() around survey::svyby() call (same as Phase 4 for svytotal)
    - tidyselect::eval_select() with allow_rename = FALSE, allow_empty = FALSE
    - Use rlang::enquo() for by parameter (same pattern as creel_design for date/strata)
    - nolint comments for cli glue variables and internal function calls
    - Reference tests with tolerance = 1e-10 comparing to manual svyby() calculations
  </implementation>
</feature>
