  <behavior>
    Performs TOST (Two One-Sided Tests) equivalence testing to determine if incomplete trip CPUE estimates are statistically equivalent to complete trip estimates within a specified threshold.

    **Cases:**

    **Input:** creel_design with trip_status field, catch column, effort column
    **Output:** creel_validation object with:
      - overall_test: TOST results for overall comparison
      - group_tests: Per-group TOST results if grouped estimation
      - equivalence_threshold: Threshold used (±20% default)
      - passed: TRUE if overall AND all groups pass equivalence
      - recommendation: "safe to use incomplete trips" or "use complete trips only"
      - metadata: sample sizes, estimates, SEs, CIs for both trip types

    **Equivalence logic:**
    - TOST tests null hypothesis: |difference| >= threshold
    - Rejects null (passes) when estimates are "close enough"
    - Two one-sided tests: (1) complete - incomplete > -threshold, (2) complete - incomplete < threshold
    - Both tests must reject (p < 0.05) for equivalence
    - Threshold: ±20% of complete trip estimate (configurable via tidycreel.equivalence_threshold option)

    **Grouped estimation:**
    - Run TOST for each group independently
    - Overall test uses ungrouped data
    - Passed only if overall AND all groups pass

    **Error cases:**
    - No trip_status field → "trip_status required for validation"
    - Insufficient complete trips (n<10) → "Need at least 10 complete trips"
    - Insufficient incomplete trips (n<10) → "Need at least 10 incomplete trips"
    - No catch/effort columns → appropriate validation error
  </behavior>

    **Technical notes:**
    - TOST requires two one-sided t-tests at alpha=0.05 each
    - Use survey-based SEs from estimate_cpue results
    - Delta method for difference variance: Var(C-I) = Var(C) + Var(I) assuming independence
    - SE_diff = sqrt(SE_complete^2 + SE_incomplete^2)
    - t = (estimate_diff - equivalence_bound) / SE_diff
    - df approximated using Welch-Satterthwaite or conservative min(n_complete-1, n_incomplete-1)
