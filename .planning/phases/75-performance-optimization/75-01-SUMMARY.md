# Plan 75-01: Profiling Harness Infrastructure — Summary

## What Was Built

Four self-contained R scripts in `inst/profiling/` that empirically profile tidycreel's computational surface:

- **00-generate-fixtures.R** — Builds realistic (30 sites, 20 days, 5 species, ~500 interviews) and stress (300 sites, 200 days, 10 species, ~5000 interviews) bus-route `creel_design` objects via the public API and saves them as RDS files to `inst/profiling/fixtures/` (gitignored — run script to recreate).
- **01-profvis-workflow.R** — Flame-graph capture for `estimate_total_catch()` and `estimate_effort()` at both scales; saves HTML artifacts to `.planning/phases/75-performance-optimization/` (requires interactive R session for htmlwidgets rendering).
- **02-bench-estimators.R** — `bench::mark()` at realistic and stress scale for `estimate_total_catch()` and `estimate_effort()`, printing scaling ratios. Wraps each benchmark in `tryCatch` for resilience.
- **03-bench-variance-methods.R** — Variance method comparison: taylor vs jackknife vs bootstrap on realistic fixture, with speedup ratios.

## Key empirical results (from 03-bench-variance-methods.R)

| Method | Median | vs taylor |
|---|---|---|
| taylor | ~1.5 ms | 1× |
| jackknife | ~28 ms | 18× |
| bootstrap | ~83 ms | 54× |

`estimate_total_catch()` scales sub-linearly: 10× data → 3.6× time (realistic → stress).

## Deviations

- `bench::mark()` in this version of `bench` does not have a `max` column; scripts use `itr/sec` instead.
- Fixture RDS files added to `.gitignore` (958 KB stress fixture exceeds git 500 KB limit).
- `bench`, `htmlwidgets`, `pkgload`, `profvis` added to `Suggests` in `DESCRIPTION` to satisfy `deps-in-desc` pre-commit hook.
- `estimate_harvest_rate()` benchmark skipped on synthetic fixture (fixture lacks species-level harvest column); its cost is captured indirectly via `estimate_total_catch()`.
