    **DESCRIPTION update (do this FIRST before any R code):**
    Add tidyselect (>= 1.2.0) to Imports in DESCRIPTION. Keep alphabetical order in Imports list.

    **Constructor behavior:**
    - creel_design(calendar, date, strata, site = NULL, design_type = "instantaneous")
    - date: single column selector (bare name or tidyselect helper) -> resolved to character name
    - strata: one or more column selectors (c(), starts_with(), etc.) -> resolved to character vector
    - site: optional single column selector -> resolved to character name or NULL
    - design_type: character string, default "instantaneous"

    **Construction pipeline:**
    1. validate_calendar_schema(calendar) -- structural check (Phase 1 validator)
    2. Resolve tidy selectors via tidyselect::eval_select() with rlang::enquo()
       - CRITICAL: enquo() MUST be called in creel_design(), NOT in helpers
       - Pass quosures to internal resolve helpers
    3. new_creel_design() -- low-level constructor, type checks via stopifnot()
    4. validate_creel_design() -- semantic Tier 1 validation

    **Internal resolve helpers (in same file):**
    - resolve_single_col(expr, data, arg_name, error_call) -> character(1)
    - resolve_multi_cols(expr, data, arg_name, error_call) -> character vector
    - Both use tidyselect::eval_select() internally
    - Both use allow_rename = FALSE, allow_empty = FALSE

    **Tier 1 validation (validate_creel_design):**
    - Date column is Date class -> cli_abort if not
    - Strata columns are character or factor -> cli_abort if not
    - Date column has no NA values -> cli_abort if has NAs
    - Site column (if provided) is character or factor -> cli_abort if not
    - Error messages use cli formatting: {.var}, {.cls}, {.code} inline styles

    **new_creel_design structure (list with class attribute):**
    ```
    list(
      calendar    = <data.frame>,
      date_col    = <character(1)>,
      strata_cols = <character>,
      site_col    = <character(1) or NULL>,
      design_type = <character(1)>,
      counts      = NULL,   # populated by add_counts() in Phase 3
      survey      = NULL    # populated internally in Phase 3
    )
    class: "creel_design"
    ```

    **Print/format/summary methods:**

    format.creel_design uses cli::cli_format_method():
    - Header: "Creel Survey Design" (cli_h1)
    - Type: design_type value
    - Date column name
    - Strata column names (comma-separated)
    - Site column name (if not NULL)
    - Calendar: N days (min_date to max_date)
    - Per-strata level counts
    - Counts status: "attached" or "none"

    print.creel_design: cat(format(x, ...), sep = "\n"); invisible(x)
    summary.creel_design: calls print(x, ...) for now; returns invisible(x)
