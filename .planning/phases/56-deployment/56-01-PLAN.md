Create `.github/workflows/pkgdown.yaml` with the following exact content (r-lib/actions v2 canonical template, adapted per locked decisions):

```yaml
# Source: https://github.com/r-lib/actions/blob/v2/examples/pkgdown.yaml
# Adapted: push to main only (no release:); workflow_dispatch included for manual redeploy
on:
  push:
    branches: [main]
  pull_request:
  workflow_dispatch:

name: pkgdown.yaml

permissions: read-all

jobs:
  pkgdown:
    runs-on: ubuntu-latest
    concurrency:
      group: pkgdown-${{ github.event_name != 'pull_request' || github.run_id }}
    env:
      GITHUB_PAT: ${{ secrets.GITHUB_TOKEN }}
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4

      - uses: r-lib/actions/setup-pandoc@v2

      - uses: r-lib/actions/setup-r@v2
        with:
          use-public-rspm: true

      - uses: r-lib/actions/setup-r-dependencies@v2
        with:
          extra-packages: any::pkgdown, local::.
          needs: website

      - name: Build site
        run: pkgdown::build_site_github_pages(new_process = FALSE, install = FALSE)
        shell: Rscript {0}

      - name: Deploy to GitHub pages
        if: github.event_name != 'pull_request'
        uses: JamesIves/github-pages-deploy-action@v4.5.0
        with:
          clean: false
          branch: gh-pages
          folder: docs
```

Key decisions (per CONTEXT.md and RESEARCH.md):
- Single combined file (not split) — build steps are identical for PR and push; `if:` guard on deploy step is the idiomatic r-lib pattern
- `workflow_dispatch:` included — allows manual redeploy without a dummy commit; low cost
- Global `permissions: read-all` with job-level override `permissions: contents: write` — minimal privilege; contents write only where needed for gh-pages push
- `concurrency:` group prevents two simultaneous deploys from racing on gh-pages; PRs each get unique run_id so they never cancel each other
- `clean: false` on JamesIves action — safer for incremental deploys; matches r-lib canonical template
- `GITHUB_PAT: ${{ secrets.GITHUB_TOKEN }}` — matches existing R-CMD-check.yaml pattern exactly
- `use-public-rspm: true` — matches existing R-CMD-check.yaml; faster package installs via Posit Public Package Manager binary repo
- No `r-version:` pin — uses R release by default (matches r-lib template; OS-independent docs output so no matrix needed)

Do NOT use:
- `pkgdown::deploy_site_github()` — deprecated; replaced by build_site_github_pages() + JamesIves action
- `peaceiris/actions-gh-pages` — not the current r-lib standard
- `clean: true` — can delete files not in current build output
- `build_site()` instead of `build_site_github_pages()` — misses .nojekyll creation and versioning awareness
