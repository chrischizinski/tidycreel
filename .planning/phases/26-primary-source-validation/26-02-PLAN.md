    Section "# Survey package cross-validation ----"

    Cross-validation approach: Manually construct a survey::svydesign() using the same
    .contribution and .pi_i columns that tidycreel computes internally, then compare
    survey::svytotal() results to tidycreel outputs. This is the same pattern established
    in Phase 24 effort estimation reference tests (test-estimate-effort.R).

    Use make_box20_6_example1() (no expansion, simpler arithmetic) for cross-validation
    since the point values are exact and the math is transparent.

    Access design internals via design$interviews which contains .pi_i and .expansion.
    Effort contribution per row: .effort_contribution (or compute as hours_fished * .expansion)
    Harvest contribution per row: fish_kept * .expansion

    Note: The exact column names in design$interviews depend on Phase 22-24 implementation.
    Per Phase 23-01 SUMMARY: add_interviews() produces .expansion = n_counted/n_interviewed
    Per Phase 24-01 SUMMARY: estimate_effort_br() uses n_counted/n_interviewed * effort / pi_i
    Per Phase 25-01 SUMMARY: estimate_harvest_br() uses fish_kept * .expansion / .pi_i

    The cross-validation builds a svydesign manually and compares:

    Tests:
    - "Effort estimate matches manual survey::svydesign + svytotal (point estimate, tol 1e-6)"
      d <- make_box20_6_example1()
      interviews <- d$interviews
      # Compute effort contribution per row (expansion * effort / 1 for HT)
      interviews$.effort_contrib <- interviews$hours_fished * interviews$.expansion
      # Manual survey design with inverse probability weights
      svy_manual <- survey::svydesign(
        ids = ~site,
        strata = ~day_type,
        weights = ~(1 / .pi_i),
        data = interviews
      )
      manual_result <- survey::svytotal(~.effort_contrib, svy_manual)
      tidycreel_result <- estimate_effort(d)
      expect_equal(
        tidycreel_result$estimates$estimate,
        as.numeric(coef(manual_result)),
        tolerance = 1e-6
      )
      # Citation: # Proves inverse probability weighting correctly implements Eq. 19.4

    - "Effort SE matches manual survey::svydesign + svytotal (tol 1e-3)"
      (same setup as above)
      expect_equal(
        tidycreel_result$estimates$se,
        as.numeric(SE(manual_result)),
        tolerance = 1e-3
      )
      # Citation: # Per CONTEXT.md: SE tolerance 1e-3 (FPC differences acceptable)

    - "Harvest estimate matches manual survey::svydesign + svytotal (point estimate, tol 1e-6)"
      d <- make_box20_6_example1()

      interviews$.harvest_contrib <- interviews$fish_kept * interviews$.expansion

        strata = ~day_type,

        data = interviews

        tidycreel_result$estimates$estimate,
