    ```yaml
            - text: Replicate Designs for Creel

    ```

    Key notes:
    - `navbar.bg: primary` tells Bootstrap to derive navbar background from the `primary` bslib variable (#1B4F72)
    - `navbar.type: dark` makes pkgdown set `data-bs-theme="dark"` on the navbar element, giving white text/links on the dark background
    - Do NOT use `.navbar-dark` CSS class — deprecated in Bootstrap 5.3 / bslib 0.6+
    - `base_font`, `heading_font`, `code_font` use bslib underscore names; `link-color`, `headings-color` use Bootstrap Sass hyphen names — both styles coexist in the bslib block
    - pkgdown automatically injects Google Fonts CDN `<link>` tags when `{google: "Name"}` syntax is used — no manual @import in extra.css needed

    ```r
    Rscript -e "pkgdown::check_pkgdown()"
    ```

    If check_pkgdown() errors on "vignettes not in articles index": add an `articles: ~` section to suppress vignette warnings. If it errors on URL mismatch: verify DESCRIPTION URL field was updated in Plan 01.

    If `link-color` or `headings-color` are rejected as unknown bslib variables, move them to extra.css as:
    ```css
    a { color: #5DADE2; }
    h1, h2, h3, h4, h5, h6 { color: #1B4F72; }
    ```
    and remove those keys from the bslib block.
