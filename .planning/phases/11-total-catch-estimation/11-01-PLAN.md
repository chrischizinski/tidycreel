**Part B: Add estimate_total_catch() to R/creel-estimates.R**

Add after estimate_harvest_grouped() (end of file). Follow exact pattern from research document Pattern 1.

Exported function `estimate_total_catch(design, by = NULL, variance = "taylor", conf_level = 0.95)`:
- Capture by_quo with rlang::enquo(by) BEFORE any validation
- Validate variance parameter (must be "taylor", "bootstrap", or "jackknife")
- Validate design is creel_design
- Call validate_design_compatibility(design)
- Route to ungrouped or grouped based on by_quo
- For grouped: resolve by_vars via tidyselect::eval_select on design$counts, call validate_grouping_compatibility, then route to grouped function
- Full roxygen2 docs with @export, @param, @return, @details, @examples, @seealso

Internal `estimate_total_catch_ungrouped(design, variance_method, conf_level)`:
- Call estimate_effort(design, variance = variance_method, conf_level = conf_level)
- Call estimate_cpue(design, variance = variance_method, conf_level = conf_level)
- Extract estimates: effort_est, cpue_est, effort_se, cpue_se
- Build named coef vector: c(effort = effort_est, cpue = cpue_est)
- Build 2x2 block-diagonal vcov matrix (zero off-diagonal = independence)
- Create svystat object: list(coefficients = coefs, vcov = vcov_matrix), class = "svystat"
- Call survey::svycontrast(stat_obj, quote(effort * cpue))
- Extract estimate via as.numeric(coef(product_contrast))
- Extract SE via as.numeric(survey::SE(product_contrast))
- Extract CI via confint(product_contrast, level = conf_level)
- Use n from cpue_result$estimates$n (interview sample size)
- Return new_creel_estimates with method = "product-total-catch"
- Mark @keywords internal @noRd

Internal `estimate_total_catch_grouped(design, by_vars, variance_method, conf_level)`:
- Call estimate_effort(design, by = !!rlang::syms(by_vars), variance = variance_method, conf_level = conf_level)
- Call estimate_cpue(design, by = !!rlang::syms(by_vars), variance = variance_method, conf_level = conf_level)
- Merge results on by_vars with suffixes c("_effort", "_cpue")
- Loop over merged rows, build per-group svystat + svycontrast
- Build result tibble with group columns + estimate + se + ci_lower + ci_upper + n
- Return new_creel_estimates with method = "product-total-catch" and by_vars set
- Mark @keywords internal @noRd

**IMPORTANT for grouped by= forwarding:** The by parameter in estimate_effort and estimate_cpue uses NSE. When calling from estimate_total_catch_grouped, use this pattern:
```r
by_syms <- rlang::syms(by_vars)
effort_result <- estimate_effort(design, by = !!by_syms, variance = variance_method, conf_level = conf_level)
```
If by_vars has length 1, use `rlang::sym(by_vars)` instead. Check test-estimate-effort.R or estimate_effort_grouped pattern for how grouped by is passed internally. If the existing grouped functions (estimate_effort_grouped, estimate_cpue_grouped) are called directly rather than going through the exported function, replicate that approach to avoid NSE issues.
