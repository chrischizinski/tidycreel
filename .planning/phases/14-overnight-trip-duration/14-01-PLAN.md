**In R/survey-bridge.R, validate_trip_metadata() function:**

Add timezone consistency check AFTER the existing POSIXct type checks for trip_start and
interview_time (around line 1033, before the "If both columns exist and are time-like" block).

The check should:
1. Extract timezone from trip_start using attr(start_vals, "tzone") (returns "" for system default)
2. Extract timezone from interview_time using attr(interview_vals, "tzone")
3. Compare: if both are non-empty AND different, add error to collection:
   - Message: "trip_start timezone '{tz1}' differs from interview_time timezone '{tz2}'. Use the same timezone for both columns."
4. If one is "" (system default) and other is explicit, that is acceptable -- R's difftime
   handles this correctly because POSIXct stores UTC internally regardless of display timezone.
   Only error when BOTH are explicitly set to DIFFERENT timezones.

Implementation note: Use `attr(x, "tzone")` not `format(x, "%Z")` because format gives
abbreviation (MDT, EDT) which varies by DST, while tzone gives the Olson name used at
construction. Empty string "" means system default timezone.

The logic:
```r
tz_start <- attr(start_vals, "tzone")
tz_interview <- attr(interview_vals, "tzone")
# Only error if both explicitly set to different timezones
if (!is.null(tz_start) && !is.null(tz_interview) &&
    nzchar(tz_start) && nzchar(tz_interview) &&
    tz_start != tz_interview) {
  collection$push(sprintf(
    "trip_start timezone '%s' differs from interview_time timezone '%s'. Use the same timezone for both columns",
    tz_start, tz_interview
  ))
}
```
