<objective>
Add `ci_method = c("delta", "bootstrap")` to the bus-route survey estimators for harvest
and catch. The implementation adds a second survey-design pass inside `br_build_estimates()`
using the existing `get_variance_design(svy, "bootstrap")` infrastructure, binding
`ci_lo_boot`/`ci_hi_boot` as extra columns in the estimates tibble when requested.

Purpose: BOOT-01, BOOT-02 — analysts can request bootstrap CIs from `estimate_total_harvest()`
and `estimate_total_catch()` without relying on delta-method normality.

Output: Updated `R/creel-estimates-bus-route.R` (br_build_estimates + internal fns),
updated `R/creel-estimates-total-harvest.R` and `R/creel-estimates-total-catch.R`
(public wrappers), new `tests/testthat/test-bootstrap-survey.R`.
</objective>
