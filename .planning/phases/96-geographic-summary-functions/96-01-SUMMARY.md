# Phase 96 Plan 01: summarize_boat_composition Summary

## One-liner

`summarize_boat_composition(design, schema)` computes mean angler-boat ratio per month x day_type from `design$counts`, returning `pct_angler_boats` in a classed data.frame.

## What Was Implemented

`summarize_boat_composition(design, schema)` appended to `R/creel-summaries.R`. Implements RPT-03: the NGPC "Angler Boats by Month and Day Type" table.

### Function behavior

- Reads `c_AnglerBoats` and `c_NonAngBoats` column names from `schema$angler_boats_col` / `schema$non_ang_boats_col`
- Fetches actual column values from `design$counts`
- Excludes count events where `angler_boats + non_ang_boats == 0`
- Computes per-row ratio `AB / (AB + NB)`
- Aggregates by month x day_type using `stats::aggregate()`: mean ratio and n_events
- Multiplies mean ratio by 100, rounds to 1 decimal for `pct_angler_boats`
- Sorts by calendar month (numeric key) then day_type; drops sort key before return
- Returns `data.frame` with class `c("creel_summary_boat_composition", "data.frame")`
- Columns: `month`, `day_type`, `n_events` (integer), `pct_angler_boats` (numeric)

### Three-guard pattern

1. Guard 1: `!inherits(design, "creel_design")` ‚Äî cli_abort with creel_design() hint
2. Guard 2: `is.null(design$counts)` ‚Äî cli_abort with add_counts() hint
3. Guard 3a: `is.null(schema$angler_boats_col)` ‚Äî cli_abort naming angler_boats_col
4. Guard 3b: `is.null(schema$non_ang_boats_col)` ‚Äî cli_abort naming non_ang_boats_col
