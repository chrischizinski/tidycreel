Expected behavior in testable terms:

**Default behavior (trip_status provided):**
- estimate_cpue(design) → uses complete trips only (use_trips = "complete" default)
- Filters to complete trips before estimation
- Uses ratio-of-means estimator by default
- Returns creel_estimates object with only complete trip interviews

**Explicit complete:**
- estimate_cpue(design, use_trips = "complete") → same as default
- Can combine with estimator="ratio-of-means" (standard, no warning)
- Can combine with estimator="mor" (valid but warns about non-standard choice)

**Explicit incomplete:**
- estimate_cpue(design, use_trips = "incomplete") → uses incomplete trips only
- Must use estimator="mor" (errors if estimator="ratio-of-means")
- Filters to incomplete trips before estimation
- Returns creel_estimates_mor object

**Backward compatibility:**
- estimate_cpue(design) when trip_status NOT provided → ignores use_trips parameter
- No errors, no warnings, perfect v0.2.0 compatibility
- Processes all interviews regardless of use_trips value

**Validation cases:**
- use_trips="incomplete" + estimator="ratio" → ERROR (strict enforcement)
- use_trips="complete" + estimator="mor" → WARNING (valid but non-standard)
- use_trips="invalid" → ERROR (invalid parameter value)
- use_trips="complete" but zero complete trips → ERROR (no trips of type)
- use_trips="incomplete" but zero incomplete trips → ERROR (no trips of type)

**Insufficient data errors:**
- use_trips="complete" but n_complete < 10 → ERROR with guidance
- use_trips="incomplete" but n_incomplete < 10 (post-truncation) → ERROR with guidance
- Error messages suggest trying diagnostic mode and referencing documentation
