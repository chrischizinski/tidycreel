    1. Rewrite .lintr with tidyverse defaults and 120-char line length (per CONTEXT.md decision: tidyverse lintr defaults):

       ```r

           line_length_linter(120)
         )
       exclusions: list(
           "data-raw",
           "inst"
         )
       ```

    2. Rewrite .pre-commit-config.yaml with the full R package hook set from lorenzwalthert/precommit (the current file only has basic pre-commit-hooks, missing all R-specific hooks):

       ```yaml
       minimum_pre_commit_version: "3.0.0"
       default_stages: [pre-commit]

       repos:
         - repo: https://github.com/lorenzwalthert/precommit
           rev: v0.4.3
           hooks:
             - id: style-files
               args: [--style_pkg=styler, --style_fun=tidyverse_style]
             - id: lintr
             - id: parsable-R
             - id: no-browser-statement
             - id: no-debug-statement
             - id: readme-rmd-rendered
             - id: use-tidy-description
             - id: deps-in-desc
         - repo: https://github.com/pre-commit/pre-commit-hooks
           rev: v4.6.0
           hooks:
             - id: trailing-whitespace
               exclude: ^(renv/|\.lintr$)
             - id: end-of-file-fixer
               exclude: ^(renv/|\.lintr$)
             - id: check-yaml
             - id: check-added-large-files
       ```

       Per CONTEXT.md and RESEARCH.md: use strict blocking (no --warn_only) since v2 starts from scratch with clean code. The style-files hook auto-fixes formatting. The lintr hook blocks commits that fail lint.

    3. Verify pre-commit is installed on the system: run `pre-commit --version`. If not installed, run `pip3 install pre-commit` or `brew install pre-commit`.

    4. Install the hooks: run `pre-commit install` from the repo root.

    5. Run a test: `pre-commit run --all-files` to verify hooks work against the current (clean) codebase. All hooks should pass since the package was just scaffolded.

    IMPORTANT: The lorenzwalthert/precommit rev version v0.4.3 is from the RESEARCH.md verified sources. If the hook install fails due to version issues, check for the latest rev at https://github.com/lorenzwalthert/precommit/releases and update accordingly.
