  <behavior>
    Two internal validator functions using checkmate + cli:

    **validate_calendar_schema(data):**
    - Input: a data frame
    - Valid when: is a data frame, has >= 1 row, has at least one Date-class column, has at least one character or factor column (for strata)
    - Invalid when: not a data frame, 0 rows, no Date column, no character/factor column
    - Returns: invisibly returns the input data frame on success
    - Errors: cli::cli_abort with collected error messages from checkmate::makeAssertCollection

    Cases:
    - data.frame(date = as.Date("2024-06-01"), day_type = "weekday") -> invisible(data) (pass)
    - data.frame(date = as.Date(c("2024-06-01", "2024-06-02")), stratum = c("A", "B"), day_type = c("weekday", "weekend")) -> invisible(data) (pass)
    - "not a data frame" -> cli_abort (fail)
    - data.frame() (0 rows) -> cli_abort (fail)
    - data.frame(x = 1:3) (no Date, no character) -> cli_abort (fail)
    - data.frame(date = c("2024-06-01", "2024-06-02"), day_type = c("weekday", "weekend")) -> cli_abort (fail, date is character not Date)
    - data.frame(date = as.Date("2024-06-01"), count = 5) -> cli_abort (fail, no character/factor column for strata)

    **validate_count_schema(data):**
    - Input: a data frame
    - Valid when: is a data frame, has >= 1 row, has at least one Date-class column, has at least one numeric (integer or double) column
    - Invalid when: not a data frame, 0 rows, no Date column, no numeric column
    - Returns: invisibly returns the input data frame on success
    - Errors: cli::cli_abort with collected error messages

    Cases:
    - data.frame(date = as.Date("2024-06-01"), count = 5L) -> invisible(data) (pass)
    - data.frame(date = as.Date("2024-06-01"), effort = 2.5, boats = 3L) -> invisible(data) (pass)
    - "not a data frame" -> cli_abort (fail)
    - data.frame() (0 rows) -> cli_abort (fail)
    - data.frame(x = "text") (no Date, no numeric) -> cli_abort (fail)
    - data.frame(date = as.Date("2024-06-01"), name = "site_a") -> cli_abort (fail, no numeric column)

    Both functions are internal (not exported) -- they use @keywords internal roxygen tag. They will be called by creel_design() and add_counts() in later phases.

    Use checkmate::assert_data_frame() for structural check.
    Check column types by iterating over columns with vapply/sapply to find Date and character/numeric columns.
    Use cli message formatting with bullet points (x for errors, i for hints).

    Pattern from RESEARCH.md:
    ```r
    validate_calendar_schema <- function(data) {
      collection <- checkmate::makeAssertCollection()
      checkmate::assert_data_frame(data, min.rows = 1, add = collection)

      if (is.data.frame(data) && nrow(data) > 0) {
        has_date <- any(vapply(data, inherits, logical(1), "Date"))
        if (!has_date) {
          collection$push("Must contain at least one Date column")
        }
        has_char <- any(vapply(data, is.character, logical(1))) ||
                    any(vapply(data, is.factor, logical(1)))
        if (!has_char) {
          collection$push("Must contain at least one character or factor column for strata")
        }
      }

      if (!collection$isEmpty()) {
        msgs <- collection$getMessages()
        cli::cli_abort(c(
          "Calendar data validation failed:",
          set_names(msgs, rep("x", length(msgs))),
          "i" = "Calendar data must be a data frame with at least one Date column and one character/factor column for strata."
        ))
      }
      invisible(data)
    }
    ```
