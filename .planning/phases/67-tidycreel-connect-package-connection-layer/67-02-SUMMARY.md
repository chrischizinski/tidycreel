## Decisions Made

- DBI status stored as `"open"` at construction and re-checked live via `DBI::dbIsValid()` in `format.creel_connection()` — ensures print always reflects actual connection state, not stale construction-time value
- CSV status is the fixed string `"ready"` — no re-validation needed since file paths don't change after construction
- Added `# nolint: object_usage_linter` on `status_str` and `padded` assignments — lintr cannot detect glue-style `{variable}` references inside `cli_format_method({})` blocks

**1. [Rule 1 - Bug] Fixed withr::local_tempdir() scoping in helper-csv.R**
- **Found during:** GREEN phase (test run after implementation)
- **Issue:** `make_test_csv()` used `withr::local_tempdir()` without `parent.frame()`, binding the temp directory's lifetime to `make_test_csv()`'s own frame. The directory was cleaned up as soon as the helper returned, before `creel_connect()` called `file.exists()`.
- **Fix:** Changed to `withr::local_tempdir(.local_envir = parent.frame())` so the temp directory lives until the `test_that()` block exits
