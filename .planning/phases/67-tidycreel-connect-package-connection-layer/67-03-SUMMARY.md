**YAML-based connection constructor with fail-fast pre-validation — all 6 CONNECT-03/04 tests green**

## Accomplishments

- Implemented `creel_connect_from_yaml(path, config = "default")` using the `config` package for multi-environment YAML support
- `.validate_yaml_config()` checks all required keys and types before any connection is attempted — aborts with `cli_abort()` listing all validation failures at once
- `.build_creel_conn()` dispatches from validated config map to `creel_connect()` (DBI or CSV backend)
- Credentials via `!expr Sys.getenv("VAR")` are evaluated at `config::get()` call time, never stored as literal strings in the object
- NAMESPACE regenerated; `creel_connect_from_yaml` exported

## Decisions Made

- `config::get(value = NULL, use_parent = FALSE)` loads the complete environment block in one call — avoids N separate `config::get(key)` calls
- Internal function renamed `.build_creel_conn` (was `.creel_connect_from_validated_config`) to avoid lintr `object_usage_linter` false positives on cross-file internal references
- `use_parent = FALSE` prevents config from searching parent directories, ensuring YAML path is authoritative
