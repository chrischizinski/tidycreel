**Create validate_trip_metadata()** (R/survey-bridge.R):

Add new internal function after validate_interviews_tier1(). Follow same pattern (checkmate::makeAssertCollection, cli::cli_abort on failure).

Parameters: `interviews`, `trip_status_col`, `trip_duration_col`, `trip_start_col`, `interview_time_col`

Validation checks (in order):

1. **trip_status_col exists** in interviews
2. **trip_status_col values are valid**: All non-NA values must be "complete" or "incomplete" (case-insensitive). Use `tolower()` for comparison.
3. **No NA in trip_status**: Error if any NA values in trip_status column.
4. **Mutually exclusive input**: Error if BOTH trip_duration_col AND (trip_start_col or interview_time_col) are non-NULL. Message: "Provide either {.arg trip_duration} or {.arg trip_start}/{.arg interview_time}, not both."
5. **trip_start requires interview_time**: Error if trip_start_col is non-NULL but interview_time_col is NULL. Message: "{.arg trip_start} requires {.arg interview_time} to calculate duration."
6. **interview_time requires trip_start**: Error if interview_time_col is non-NULL but trip_start_col is NULL.
7. **Duration column validation** (if trip_duration_col provided):
   - Column exists in interviews
   - Column is numeric
   - No NA values (error: "trip_duration is required for all interviews")
   - No negative values (error)
   - No values < 1/60 hours (1 minute) (error: "Trip durations less than 1 minute are unrealistic")
   - Warn if any values > 48 (cli::cli_warn, not error)
8. **Time column validation** (if trip_start_col and interview_time_col provided):
   - Columns exist in interviews
   - Columns are POSIXct or POSIXlt (error if not time-like)
   - No NA values in either column
   - Computed duration (interview_time - trip_start) must not be negative (error)
   - Computed duration must not be < 1/60 hours (error)
   - Warn if any computed duration > 48 hours

Use @keywords internal @noRd. Return invisible(NULL) on success (validation is side-effect only, unlike validate_interviews_tier1 which returns creel_validation).
