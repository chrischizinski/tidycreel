## Tech Debt

### Phase 84 — Camera Missing Data

| Item | Severity | Description |
|------|----------|-------------|
| CR-01 | Advisory | `.imputed` flag is set based on post-imputation inference (`status != "operational" & !is.na(count)`) rather than a pre-imputation NA snapshot. Non-operational rows with pre-existing non-NA counts will be incorrectly flagged as imputed. Typical use case (outage = NA) is unaffected. |
| CR-02 | Advisory | Function docs say "zero-inflated negative binomial GLMM" but `glmmTMB::nbinom2(link = "log")` without `ziformula` is a plain NB GLMM. Documentation inaccuracy — no functional impact, but misleads users and paper citations. |

### Phase 85 — Mark-Recapture Estimators

| Item | Severity | Description |
|------|----------|-------------|
| WARNING-01 | Advisory | `variance_method = "chapman"` label stored in Petersen branch output — should be `"delta"` or `"petersen"`. No functional impact on estimates. |
| WARNING-02 | Advisory | Schnabel `ci_hi <- sum_Mn / lo_m` is unguarded when `lo_m = 0` (`qpois(0.025, 1) = 0`). Produces `Inf` silently. Biologically unrealistic inputs (sum_m < 3) are affected. |
| WARNING-03 | Advisory | `estimate_mr_harvest()` upper-bound guard (`harvest_rate > 1`) exists in code (line 280) but Test J does not exercise it. Guard is functional but untested. |
