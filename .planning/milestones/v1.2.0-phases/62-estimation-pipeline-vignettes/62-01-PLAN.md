## Introduction
Brief framing paragraph: what this vignette covers and who it is for. Audience is a creel biologist who knows surveys but not sampling theory. State that PSUs, two-stage variance, and the HT estimator will all be explained from first principles.

## What is a PSU?
Explain the Primary Sampling Unit in plain language: in an instantaneous-count creel survey, each day × stratum combination is a PSU. Show a tiny inline calendar (3 days × 2 strata = 6 PSUs) as a data.frame. No R code needed yet — just the table.

## The Basic Effort Estimator
Single-count-per-day case. Show the Horvitz–Thompson form:
$$\hat{E} = \sum_{i \in s} \frac{y_i}{\pi_i}$$
Gloss each symbol: $y_i$ = observed count on PSU $i$; $\pi_i$ = sampling probability for PSU $i$; $s$ = sample of PSUs selected.

Then show the practical creel form (no fractional probabilities — strata with N_h total days and n_h sampled days):
$$\hat{E}_h = \frac{N_h}{n_h} \sum_{i \in s_h} y_i \times T_i$$
where $T_i$ = open hours on day $i$. Gloss each symbol. Include a worked numeric: stratum "weekday" with N=10, n=3, counts=[15, 23, 19], T=[10, 10, 10]. Compute $\hat{E}_{\text{weekday}}$ by hand. Then verify with a minimal R code block using creel_design + add_counts + estimate_effort.

## Multiple Counts per Day: Rasmussen Two-Stage Variance
Explain the motivation: when observers count the same PSU twice (morning + afternoon), we have replicated measurements. The within-PSU replication gives us extra information about measurement variance that a single-count design cannot separate from between-PSU variance.

Show the two-stage variance decomposition. Display as annotated LaTeX:
$$\text{SE}^2(\hat{E}) = \text{SE}^2_{\text{between}} + \text{SE}^2_{\text{within}}$$

Rasmussen (1994) between-day component:
$$\text{SE}^2_{\text{between}} = \frac{N_h^2}{n_h} \left(1 - \frac{n_h}{N_h}\right) s^2_{\bar{y}}$$
where $s^2_{\bar{y}}$ is the sample variance of the within-day means $\bar{y}_i$.

Within-day component:
$$\text{SE}^2_{\text{within}} = \frac{N_h^2}{n_h^2} \sum_{i=1}^{n_h} \frac{s^2_{ij}}{m_i}$$
where $s^2_{ij}$ is the sample variance of counts on day $i$ and $m_i$ is the number of counts on that day.

After each equation, add a plain-language gloss sentence.

Show a worked numeric table with columns: day | count_1 | count_2 | within-day mean | se_between contribution | se_within contribution. Use 4 days with round numbers. Compute by hand below the table, then confirm with an R code block using estimate_effort() on the same data. The R output's se_between and se_within must match the by-hand values shown.

## The Progressive Count Estimator
Explain as a distinct section (not a subsection of multiple counts): in a progressive count, the observer traverses the entire section and records every angler encountered, rather than taking a snapshot. The estimator is mechanically different.

Show the progressive count formula:
$$\hat{E}_d = C_d \times T_d$$
where $C_d$ = count on circuit on day $d$ and $T_d$ = total open hours. Gloss symbols. Explain WHY the formula is $C \times T$: the observer's travel time $\tau$ cancels because effort is anglers × hours, and a progressive count records anglers present per unit travel time, scaled by total time.

Include a minimal worked numeric (3 days, round numbers) and confirm with an R code block using the progressive_count = TRUE pathway in estimate_effort().

## Summary
One paragraph summarising: single-count → Rasmussen two-stage → progressive count, and when each is appropriate.
