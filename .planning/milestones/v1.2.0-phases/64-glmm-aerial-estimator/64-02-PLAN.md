Section order (locked: decision guide must come first):

**1. When to Use the GLMM Estimator** (decision guide — 2-3 paragraphs, no code)

Opening paragraph: Explain the core bias problem. "If your pilot always flies at the same time of day — say, 10 AM — your instantaneous count systematically over- or under-represents total daily effort. Early-morning flights catch fewer anglers than are present at peak hours; late-afternoon flights may miss early starters entirely. When flight timing is non-random, the simple expansion (count × h_open / v) inherits this temporal bias."

Second paragraph: Explain the GLMM fix. "The GLMM approach (Askey 2018) models how angler counts change through the day using a quadratic hour effect and a day-level random intercept: count ~ poly(hour, 2) + (1|date). Once the diurnal curve is estimated, the model integrates predicted counts across the full open-water window — correcting for wherever in the curve the actual flight fell."

Third paragraph (decision guide table or bullet list): "Use the GLMM estimator when: (1) flights always occur in the same part of the day (fixed morning or afternoon schedule), (2) you have multiple overflights per day across several days (minimum 8-10 survey days recommended for stable random-effect estimation). Use the simple estimator when: (1) flight timing is randomly assigned across the open-water window, (2) you have only one count per day."

Cross-link: "For the basic aerial workflow without GLMM correction, see the [aerial surveys vignette](aerial-surveys.html)."

**2. Example Data**

Brief paragraph: describe example_aerial_glmm_counts (12 days, 4 flights per day at 7, 10, 13, 16 hours). Show structure with head(example_aerial_glmm_counts) and a glimpse or table.

**3. Building the Aerial Design**

Code chunk: construct aerial creel_design from example_aerial_glmm_counts dates/day_type, survey_type = "aerial", h_open = 14. add_counts(design, example_aerial_glmm_counts).

**4. GLMM Effort Estimation**

Code chunk: call estimate_effort_aerial_glmm(design, time_col = time_of_flight). Assign to glmm_result. Print result.

Brief text: explain what the model fits (negative binomial GLMM, quadratic hour, day random intercept), what the integration does (sum predicted counts across the open-water window, scale by h_open/v).

**5. Variance and Confidence Intervals**

Explain the two SE options. Default: delta method (propagates fixed-effect covariance to the derived total; fast, analytic). Optional: parametric bootstrap via lme4::bootMer (more accurate for skewed count distributions; slower).

Code chunk showing bootstrap path: estimate_effort_aerial_glmm(design, time_col = time_of_flight, boot = TRUE, nboot = 100). Use nboot = 100 in vignette for fast rendering (note in text: "Production analyses use the default nboot = 500").

**6. Comparison: Simple vs. GLMM Estimator**

This is the side-by-side bias comparison (locked requirement). Run both estimators on the same design and show results together.

Code chunk:
- simple_result <- estimate_effort(design) (uses simple aerial path)
- glmm_result already computed above
- Build a comparison tibble: bind_rows(mutate(glmm_result$estimates, method = "GLMM"), mutate(simple_result$estimates, method = "Simple")) and select/print estimate, se, ci_lower, ci_upper, method

Explanatory text: "The GLMM estimate corrects for the fact that all flights occurred at fixed hours (7, 10, 13, 16). The simple estimator treats each count as representative of the full open-water window, which inflates or deflates the total depending on where the peak falls in the diurnal curve."

**7. Custom Formula**

Short section showing formula override: estimate_effort_aerial_glmm(design, time_col = time_of_flight, formula = n_anglers ~ time_of_flight + (1|date)) for users who want a linear rather than quadratic temporal term.
