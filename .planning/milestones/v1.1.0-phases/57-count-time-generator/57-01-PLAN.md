    ```r
    generate_count_times <- function(
      start_time,
      end_time,
      strategy,
      n_windows  = NULL,
      window_size = NULL,
      min_gap    = NULL,
      fixed_windows = NULL,
      seed       = NULL
    )
    ```

    **Validations (in order, all via cli_abort()):**

    1. `strategy` must be supplied (no default) and must be one of `c("random", "systematic", "fixed")`. Use explicit `%in%` check — same pattern as `survey_type` in `creel_design()`. Error header: "Unknown strategy."

    2. When `strategy != "fixed"`: validate `start_time` and `end_time` match `"^[0-2][0-9]:[0-5][0-9]$"` via `grepl()`. Error: "start_time and end_time must be in HH:MM format."

    3. When `strategy != "fixed"`: convert to integer minutes via `parse_hhmm_to_min()`. Enforce `end_min > start_min`. Error: "end_time must be after start_time."

    4. When `strategy != "fixed"`: `n_windows` and `window_size` and `min_gap` must all be non-NULL integers. Error per missing arg using `cli_abort()` with `call = rlang::caller_env()`.

    5. When `strategy != "fixed"`: `total_min %% n_windows != 0` — error: "{total_min} minutes / {n_windows} windows = {total_min/n_windows} — span must divide evenly; adjust n_windows or start/end time."

    6. When `strategy != "fixed"`: compute `k <- total_min / n_windows`. If `window_size + min_gap > k` — error: "stratum is {k} min, window_size + min_gap = {window_size + min_gap} min — reduce n_windows, window_size, or min_gap."

    **Random strategy logic (inside `withr::with_seed(seed, { ... })`):**
    - For stratum `i` (0-indexed): stratum range is `[start_min + i*k, start_min + i*k + k - window_size]`
    - Sample `t_start_i` from this range using `sample(stratum_start:(stratum_start + k - window_size), 1)`
    - `t_end_i <- t_start_i + window_size`

    **Systematic strategy logic (inside `withr::with_seed(seed, { ... })`):**
    - Sample `t1` from `[start_min, start_min + k - window_size]` using `sample(start_min:(start_min + k - window_size), 1)`
    - Compute `t_start_i <- t1 + (i - 1) * k` for `i` in `1:n_windows`
    - `t_end_i <- t_start_i + window_size`
    - Verify all windows within bounds (should be guaranteed by construction — add defensive stopifnot)

    **Fixed strategy logic:**
    - Validate `fixed_windows` is a data frame with `start_time` and `end_time` columns. Error if not.
    - Convert both columns to minutes, sort by `start_time`.
    - Check non-overlapping: for consecutive rows, `end_min[i] <= start_min[i+1]`. If any overlap: cli_abort() naming the overlapping rows.
