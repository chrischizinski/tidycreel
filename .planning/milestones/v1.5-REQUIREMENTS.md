# Requirements: tidycreel v1.5.0

**Defined:** 2026-04-26
**Core Value:** A biologist should be able to go from survey design to package-ready estimates, plots, summaries, and documentation without stitching together a custom analysis stack.

## v1.5.0 Requirements

### Estimators

- [x] **ESTIM-01**: `estimate_total_catch()` returns the same aggregate estimate as summing `estimate_total_catch(by = "species")` for multi-strata designs (INV-06 fix — stratified-sum) — **Validated v1.5.0** (stratified-sum product estimator; INV-06 quickcheck 24/24 pass)
- [x] **ESTIM-02**: `estimate_exploitation_rate()` accepts tagged-fish count and creel-harvest data and returns an exploitation rate using the Pollock et al. formulation — **Validated v1.5.0** (standalone scalar estimator, delta-method SE, [0,1]-clamped CI)
- [x] **ESTIM-03**: `estimate_exploitation_rate()` supports stratification by survey strata and returns stratum-level estimates — **Validated v1.5.0** (T-weighted aggregate .overall row via internal helper)
- [x] **ESTIM-04**: INV-06 quickcheck property test passes for multi-strata multi-species designs after fix — **Validated v1.5.0** (24/24 invariant tests pass including multi-strata quickcheck)
- [x] **ESTIM-05**: `estimate_exploitation_rate()` is documented with a worked example in its Rd file — **Validated v1.5.0** (self-contained @examples, pkgdown reference page confirmed)
