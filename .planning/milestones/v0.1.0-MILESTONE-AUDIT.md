## Technical Debt

### Accepted Deviations

#### Core Estimation Coverage (TEST-09)

**Impact:** Low
**Phase:** 7 (Quality Assurance)
**Status:** Accepted as pragmatic architectural decision

**Description:**
Core estimation files (R/creel-estimates.R and R/survey-bridge.R) average 84.49% coverage, below the 95% target. The coverage gaps consist entirely of defensive error handlers in R/survey-bridge.R that cannot be reached through normal API usage due to earlier validation layers (Tier 1 at design creation).

**Examples of unreachable code:**
- Lonely PSU error handling (caught during estimation, but Tier 1 validates PSU structure)
- Missing column errors in internal functions (caught by tidyselect::eval_select before reaching internal code)
- Invalid design structure checks (impossible to reach via public API)

**Rationale for acceptance:**
1. Testing these paths requires bypassing public APIs and calling internal functions directly
2. Such tests would be brittle and test implementation details rather than user behavior
3. The defensive code provides safety but is unreachable in production due to layered validation
4. Overall package coverage (88.75%) exceeds the 85% target
5. All user-facing behavior is comprehensively tested

**Documented in:** 07-02-SUMMARY.md decision: "accept-unreachable-error-handlers"

**Recommendation:** No action required for v0.1.0 release.
