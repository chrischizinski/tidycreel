## Milestone Summary

**Key Decisions:**

- `n_counted`/`n_interviewed` are not raw API fields — implemented `bank_anglers/angler_boats/non_ang_boats` schema instead (API-11 plan divergence, confirmed via live NGPC probe)
- `format.creel_connection` API branch added to `print-methods.R` — shows `auth$type` only, never token; protective comment guards against future regression
- Bootstrap for bus-route/ice uses `survey::as.svrepdesign()` via `get_variance_design(svy, "bootstrap")` — cost is in upstream survey package, not addressable in tidycreel
- `estimate_total_harvest()` gained a bus-route dispatch block (previously missing) in Phase 94-01
- `isNamespaceLoaded()` guard pattern canonicalized for validation scripts sourced during R CMD check
- ASCII `--` replaces Unicode em dashes in R source comments (U+2014 only works in string literals, not comments)
- `generics` package used for `tidy()` dispatch (not broom directly) to avoid heavy dependency

**Issues Resolved:**

- SEC-01: `print-methods.R` had no API branch — fell through to DBI path showing wrong label
- API-09: 6 TODO stubs in `creel-discovery.R` api_rename_map confirmed via NGPC live probe
- API-10: `list_creels()` early return used `data.frame()` not `tibble::tibble()` — 0-column result
- QUAL-02: 30+ non-ASCII em dashes in tidycreel.connect source caused rcmdcheck WARNING
- QUAL-01: validation script called `devtools::load_all()` unconditionally, corrupting S3 dispatch under R CMD check
- Phase 91-03 schema divergence: n_counted/n_interviewed not raw API fields; full bank_anglers/angler_boats/non_ang_boats refactor across both packages

**Tech Debt Incurred:**

- EXPORT-02 xlsx path has no test (code exists, guarded with check_installed; pattern from write_schedule SCHED-03 not applied)
- No VERIFICATION.md files created for Phases 91-94 (process gap; evidence in SUMMARY.md + integration checker)
- REQUIREMENTS.md API-11 description named wrong columns at close (fixed during milestone close)

**UAT Results:** 18/18 tests passed (17 passed directly; 1 minor issue — tidycreel.connect URL/title/email metadata — fixed in commit de96d57)
