**Status:** ✅ SHIPPED 2026-05-11
**Phases:** 88–90

## Overview

Completed the `tidycreel.connect` API backend so users can fetch creel data from any compatible REST service, and validated the full estimation pipeline against known real-world outputs from the Calamus 2016 archive.

## Phases

### Phase 88: httr2 Hardening and API Fetch Methods

**Goal**: Users can call any `fetch_*` method on a `creel_connection_api` object and receive a canonical data frame, backed by an HTTP layer that surfaces structured errors and retries on transient failures
**Depends on**: Phase 87 (stable v1.6.0 baseline; existing `.api_fetch()` infrastructure in `tidycreel.connect`)
**Requirements**: API-01, API-02, API-03, API-04, API-05, API-06
**Plans**: 3 plans

Plans:
- [x] 088-01-PLAN.md — Promote httr2 to Imports, harden `.api_fetch()` with `req_error`/`req_retry`/`cli_abort`, write helper-api.R and test-api-fetch.R (API-06)
- [x] 088-02-PLAN.md — Add `fetch_interviews.creel_connection_api` and `fetch_counts.creel_connection_api` with tests (API-01, API-02)
- [x] 088-03-PLAN.md — Add `fetch_catch`, `fetch_harvest_lengths`, `fetch_release_lengths` API methods with tests (API-03, API-04, API-05)

**Details:**

Plan 088-01: Promoted `httr2` from `Suggests` to `Imports` in `tidycreel.connect/DESCRIPTION` with floor `>= 1.0.0`. Rewrote `.api_fetch()`: removed `requireNamespace("httr2")` guard; added `req_error(req, is_error = \(resp) FALSE)` (disable httr2 auto-abort); added `req_retry(req, max_tries = 3L, is_transient = \(resp) httr2::resp_status(resp) %in% c(429L, 503L))` for transient-failure retry; added `status >= 400L` branch with `cli_abort` including HTTP status and API error body. Created `helper-api.R` fixture and `test-api-fetch.R` with 4 mocked test cases (200 OK, 404 error, 429→200 retry success, 429×3 abort).

Plan 088-02: Implemented `fetch_interviews.creel_connection_api` and `fetch_counts.creel_connection_api`. Added `.rename_api_to_canonical()` helper. Established canonical pattern: `.api_fetch()` → early 0-row typed return on `[]` → hardcoded `api_rename_map` rename → effort arithmetic (hours + minutes/60) → `.parse_api_date()` ISO-8601 fix → validate. Key NGPC field names: `ii_UID → interview_uid`, `cd_Date → date`, `Num → catch_count`, `ii_TripType → trip_status`, `ii_NumberAnglers → angler_count`.

Plan 088-03: Implemented `fetch_catch`, `fetch_harvest_lengths`, `fetch_release_lengths` for `creel_connection_api`. `catch_uid` and `length_uid` synthesized via `seq_len(nrow(df))` (absent from API). `length_type` injected as constant. Critical pitfall: `iiUID` (no underscore) is harvest/release length join key vs `ii_UID` (with underscore) for interviews — wrong map silently drops `interview_uid`. Species coerced to character (API may return integer codes like `86`).

## Milestone Summary

**Key Decisions:**
- `httr2` promoted to `Imports` (floor `>= 1.0.0`) — required for structured retry/error without runtime guards
- `req_retry` must be called BEFORE `req_error`; httr2 1.2.2 short-circuits `is_transient` on status ≥ 400 unless explicit predicate provided
- API field names are NGPC-fixed — each `fetch_*` method uses hardcoded `api_rename_map`, never schema key lookups
- `iiUID` (no underscore) is harvest/release length join key; `ii_UID` (with underscore) for interviews — critical distinction
- `local_mocked_responses()` intercepts before retry loop — positive retry path (429→200) cannot be unit-tested with httr2 mocks; human verification with real/staging server deferred
- `search_creels()` is client-side — calls `list_creels(conn)` then `tolower()` + `grepl(fixed = TRUE)` filter
- R silently drops `ignore.case` when `fixed = TRUE` — use `tolower()` + `fixed = TRUE`, NOT `ignore.case + fixed = TRUE`
- `estimate_harvest_rate(design)` used for bus-route harvest total (not `estimate_total_harvest()`) — dispatches to Jones & Pollock Eq. 19.5

**Issues Resolved:**
- httr2 retry pitfall identified and worked around (documented in MEMORY)
- grepl metacharacter crash fixed in `search_creels()` (Phase 89 late-stage fix)
- Calamus 2016 data characteristics (duplicated UIDs, three catch_types, species code 86) handled without silent errors

**Issues Deferred (carry-forward to v1.8.0):**
- NGPC discovery field names unconfirmed — all 6 are TODO stubs in `list_creels.creel_connection_api`
- Discovery endpoint path unconfirmed (`"AnalysisData/GetAvailableCreels"`)
- `list_creels()` silent 0-column return when rename map fields all miss (WR-01 not fixed)
- `@param endpoints` doc omits `"discovery"` key (WR-02 not fixed)
- Bus-route API E2E gap: `fetch_interviews` returns only 5 canonical columns; missing `n_counted`/`n_interviewed` for `add_interviews()` expansion factor
- Validation script lacks working-directory guard
- Pre-existing rcmdcheck warnings in tidycreel.connect (non-ASCII in creel-connect-yaml.R, missing VignetteBuilder)

**Technical Debt Incurred:**
- All discovery rename maps are placeholder stubs — will silently produce 0-column data frame against real NGPC API until confirmed field names are wired in
