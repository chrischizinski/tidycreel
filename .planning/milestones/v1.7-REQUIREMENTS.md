# Requirements Archive: tidycreel v1.7.0

**Archived:** 2026-05-16
**Milestone:** v1.7.0 — API Connection & Real-Data Validation
**Status:** ✅ COMPLETE — all 9 requirements satisfied
**Core Value:** A biologist should be able to go from survey design to package-ready estimates, plots, summaries, and documentation without stitching together a custom analysis stack.

## v1.7.0 Requirements

### API — Fetch Methods

- [x] **API-01**: User can call `fetch_interviews()` on a `creel_connection_api` object and receive a data frame with canonical columns (`interview_uid`, `date`, `catch_count`, `effort`, `trip_status`) — satisfied Phase 88
- [x] **API-02**: User can call `fetch_counts()` on a `creel_connection_api` object and receive a data frame with canonical columns (`date`, `angler_count`) — satisfied Phase 88
- [x] **API-03**: User can call `fetch_catch()` on a `creel_connection_api` object and receive a data frame with canonical columns (`catch_uid`, `interview_uid`, `species`, `catch_count`, `catch_type`) — satisfied Phase 88
- [x] **API-04**: User can call `fetch_harvest_lengths()` on a `creel_connection_api` object and receive a data frame with canonical columns (`length_uid`, `interview_uid`, `species`, `length_mm`, `length_type = "harvest"`) — satisfied Phase 88
- [x] **API-05**: User can call `fetch_release_lengths()` on a `creel_connection_api` object and receive a data frame with canonical columns (`length_uid`, `interview_uid`, `species`, `length_mm`, `length_type = "release"`) — satisfied Phase 88
- [x] **API-06**: `.api_fetch()` returns a structured, human-readable error when the API returns a non-2xx response and retries up to 3 times on transient failures (429/503) — partially satisfied Phase 88 (abort-on-3×429 verified; positive retry path is architectural constraint — httr2 1.2.2 mock architecture prevents unit-testing)

### API — Discovery

- [x] **API-07**: User can call `list_creels()` on a `creel_connection_api` object and receive a data frame of all available surveys with columns `creel_uid`, `title`, `description`, `active`, `data_complete`, `comments` — satisfied Phase 89 (field names are TODO stubs pending live API confirmation)
- [x] **API-08**: User can call `search_creels()` on a `creel_connection_api` object with a keyword string and receive matching surveys in the same shape as `list_creels()` — satisfied Phase 89

### Real-Data Validation

- [x] **REAL-01**: A standalone integration script in `inst/validation/` runs the full bus-route estimation pipeline on Calamus 2016 archived data and reports whether effort, catch, and harvest estimates match the archived reference comparison outputs within acceptable tolerance — satisfied Phase 90 (all 3 estimands within 0.1%)

## Out of Scope

| Feature | Reason |
|---------|--------|
| `fetch_*` for `creel_connection_sqlserver` | Phase 69 stub territory; separate milestone |
| Code table accessors (species_codes, waterbody_codes, etc.) | Not needed for estimation pipeline; anti-feature per research |
| Auto-discovery of UIDs in `creel_connect_api()` constructor | Anti-feature — forces network call on construction; breaks offline workflows |
| Automatic species code join on every fetch | Anti-feature — imposes opinionated recoding on raw data; user controls this |
| testthat snapshot tests for Calamus 2016 data | User selected integration script over test-suite approach |
| Jolly-Seber open-population estimator | Output contract incompatible with `creel_estimates`; requires new S3 class |
