### Phase 81: Exploitation-Rate Estimator

**Goal**: Biologists can call `estimate_exploitation_rate()` with tagged-fish counts and creel-harvest data and receive stratum-level and aggregate exploitation-rate estimates documented with a worked example
**Depends on**: Phase 80 (clean estimator baseline)
**Requirements**: ESTIM-02, ESTIM-03, ESTIM-05
**Plans**: 3 plans

Plans:

- [x] 081-01: Implement `estimate_exploitation_rate()` unstratified path with TDD (ESTIM-02)
- [x] 081-02: Add stratified path and complete Roxygen docs with @examples (ESTIM-03, ESTIM-05)
- [x] 081-03: Integration gate — quickcheck invariants, rcmdcheck, pkgdown reference (ESTIM-05)

**Details:**

Standalone scalar estimator pattern: accepts pre-computed summary stats (T, C, se_C, n, m) with no `creel_design` dependency. Delta-method variance: `Var(u) = r² × p(1-p)/n + p² × se_C²/T²`. Stratified path via internal `.estimate_exploitation_rate_stratified()` helper with T-weighted aggregate `sum(T_h × u_h) / sum(T_h)`. Four quickcheck invariants (bound, monotone-C, monotone-m, strata-agg). pkgdown Estimation section updated. 2537 tests passing.

## Milestone Summary

**Key Decisions:**

- Stratified-sum product estimator: per-stratum `E_h × CPUE_h` summed, not pooled `E × CPUE`
- `cpue_for_stratum_product()` helper filters complete trips before `estimate_cpue_grouped()` to avoid NSE issues
- `estimate_exploitation_rate()` takes scalar inputs — no `creel_design` object needed (pre-computed summary stats)
- Stratified path as internal `.estimate_exploitation_rate_stratified()` — router guard at top of main function
- quickcheck API: `for_all()` / `integer_bounded(left=, right=)` / `double_bounded(left=, right=)` (not gen_int/gen_dbl/forall)
- `@importFrom lifecycle badge` placed in `R/tidycreel-package.R` (single package-level declaration)
- goodpractice T/F finding deferred: parameter `T` is canonical Pollock et al. domain notation; renaming breaks public API
- rhub v2 (GitHub Actions-based) selected over legacy rhub v1 for auditable cross-platform checks

**Issues Resolved:**

- INV-06: `estimate_total_catch()` combined-ratio inconsistency across multi-strata designs
- rcmdcheck NOTE for unused `lifecycle` import
- 14 `sapply()` calls producing untyped output replaced with `vapply()`

**Issues Deferred:**

- QUAL-05: rOpenSci software review submission — deferred to undetermined future date
- goodpractice cyclomatic complexity (creel_design CC=74, estimate_catch_rate CC=66) — requires estimator core refactor
- goodpractice line-length (781+ instances) — mass reformatting deferred
- coverage increase above 86% — at threshold; major test additions out of scope

**Technical Debt Incurred:**

- goodpractice T/F finding: `T` parameter in `estimate_exploitation_rate()` shadows `TRUE`; renaming requires breaking API change
- Windows platform not included in rhub check run; only Linux and macOS verified
