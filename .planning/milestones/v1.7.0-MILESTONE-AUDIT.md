## Tech Debt by Phase

### Phase 88 (3 items)

1. **API-06 positive retry (429→200)** — httr2 1.2.2 `local_mocked_responses()` intercepts before retry loop; `is_transient` predicate wired correctly but positive path is only verifiable with a real/staging server. Abort-on-3×429 path verified. Requires human confirmation before true completeness.

2. **VALIDATION.md draft** — `nyquist_compliant: false, wave_0_complete: false`. Nyquist validation was started but never completed for Phase 88.

3. **bus_route API E2E gap** — `fetch_interviews.creel_connection_api` deliberately returns only the 5 columns in API-01 spec. A user running a bus_route survey design via API will find `n_counted` and `n_interviewed` absent, causing `add_interviews()` to fail. API-01 as stated is met; the broader bus_route API E2E flow is not. Needs explicit documenting or a follow-on requirement.

### Phase 89 (5 items)

4. **All discovery field names are TODO stubs** — `cr_UID`, `Creel_Name`, `sr_Title`, `Active`, `DataComplete`, `sr_Comments` in `list_creels.creel_connection_api` api_rename_map. All have `# TODO: confirm field name with live API`. Silent 0-column return risk (WR-01 not fixed).

5. **Discovery endpoint path unconfirmed** — `"AnalysisData/GetAvailableCreels"` at creel-connection-api.R:140 has TODO comment.

6. **WR-01 not fixed** — `list_creels()` returns 0-column data frame silently when all rename-map fields fail; no `cli_warn()` guard added.

7. **WR-02 not fixed** — `@param endpoints` in `creel_connect_api()` docs omits `"discovery"` key.

8. **SUMMARY frontmatter missing `requirements:` field** — 089-01 and 089-02 SUMMARY.md files lack `requirements: [API-07, API-08]`; REQUIREMENTS.md traceability not updated at phase close.

### Phase 90 (2 items)

9. **Working-directory guard absent** — `calamus-2016-validation.R` uses relative path `inst/extdata/calamus-2016` with no directory existence check; opaque errors if not run from package root (W-01 from 90-REVIEW.md).

10. **Discovery field names not validated against live API** — Phase 90 implemented as offline-only CSV validation; original scope (per Phase 88 CONTEXT.md) included confirming NGPC API field names. TODO stubs in creel-discovery.R remain the only spec for discovery field names.

### Milestone-Wide (2 items)

11. **REQUIREMENTS.md stale** — Checkboxes `[ ]` for API-01 through API-08; traceability table shows "Pending" for these 8 requirements. Cosmetic (code is complete) but misleading.

12. **Pre-existing rcmdcheck warnings in tidycreel.connect** — Non-ASCII in `creel-connect-yaml.R`; missing `VignetteBuilder`. Pre-date v1.7.0; logged in Phase 89 `deferred-items.md`.
