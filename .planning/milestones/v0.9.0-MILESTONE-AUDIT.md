## E2E User Flow Verification

### Flow 1: Pre-season planning (SCHED-01/02 + POWER-01/02 + VALID-01)
```
generate_schedule(season_start, season_end, ...) → creel_schedule
generate_bus_schedule(schedule, ...) → tibble with inclusion_prob
creel_n_effort(cv_target, N_h, ybar_h, s2_h) → n_required per stratum
validate_design(creel_design(...), cv_target, n_min) → creel_design_report
```
**Status:** ✓ Wired. `validate_design()` internally calls `creel_n_effort()` and `creel_n_cpue()` (lines 68, 81). Bus schedule `inclusion_prob` column feeds `creel_design()` directly.

### Flow 2: Post-season quality check (QUAL-01)
```
creel_design(bus_schedule, ...) → creel_design object
check_completeness(design, interviews, counts, n_min) → creel_completeness_report
```
**Status:** ✓ Wired. `check_completeness()` dispatches by `survey_type`; aerial/camera skip interview warnings via `intersect()` guard.

### Flow 3: Season reporting (REPT-01)
```
list(effort = creel_estimates, cpue = creel_estimates, ...) |>
  season_summary() → creel_season_summary (wide tibble)
```
**Status:** ✓ Wired. Each list element validated as `creel_estimates`; `by_vars` consistency enforced; `bind_cols` (no strata) or `Reduce(left_join)` (with strata) handles both cases.

### Flow 4: Power/sample-size (POWER-01–04)
```
creel_n_effort() / creel_n_cpue() → required n
creel_power_cpue() → statistical power given alpha, n, historical CV
creel_cv_effort() / creel_cv_cpue() → expected CV given known n
```
**Status:** ✓ Verified. Pure numeric functions; no S3 dependency chain; correctly called from `validate_design()`.

### Flow 5: Schedule I/O round-trip (SCHED-03/04)
```
write_schedule(schedule, path, format = "csv") → file
read_schedule(path) → creel_schedule (validated)
```
**Status:** ✓ Wired. `new_creel_schedule()` constructor wraps data.frame; `read_schedule()` re-applies class and validates column types.
